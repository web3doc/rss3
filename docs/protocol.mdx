# 协议

项目链接：<https://github.com/NaturalSelectionLabs/RSS3>

我们已经看到了 Web1 时代，每个人都乐于将 RSS 用作联合的标准格式，而 Web2 时代，RSS 衰落并创建了平台之间的信息封闭。

我们都同意在 Web3 时代信息应该更加自由地分发，为了促进这一点，我们需要一种我们都可以用来联合信息的通用格式，从而降低使用和开发成本。所以我们创建了一个新的协议RSS3，RSS代表对历史的致敬，3代表Web3。

RSS3 是为统一信息格式和促进信息自由流动而创建的协议。

## 最新草稿

当前版本：**v0.3.1**

RSS3在用户端以文件的形式存在。 RSS3 文件有两种类型 - RSS3Index 和 RSS3List，每个角色生成的文件都由角色签名。

索引文件是以角色的以太坊地址命名的入口文件，它定义：

- `items`：将被构建到提要中的角色的活动。自定义项目是角色在 RSS3 上本地发布的内容，而自动项目是来自不同网络的活动（例如，转移 NFT）
- `profile`：角色的姓名、头像、简历和经过验证的帐户
- `assets`：资产是由节点收集的角色所拥有的数字资产，例如 NFT。支持自行声明的自定义资产。
- `links`：链接是与其他对象的关系
